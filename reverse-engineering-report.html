Section 1: Project Overview

What This Project Does

AI-powered terminal platform with a daemonized agent runtime, multiple UIs (Ink terminal, Web, FieldView), pluggable tool ecosystem, and multi-provider model support. It mediates user prompts through a tool-aware agent loop and streams structured events to clients over WebSockets.

Languages: TypeScript, React

Frameworks: Ink, Vite (web), WebSocket

AI Providers: OpenRouter, Anthropic, Z.ai, MiniMax, LM Studio

Entry Points

  • CLI bin: src/bin/ff-terminal.ts
  • Daemon: src/daemon/daemon.ts
  • Ink UI: src/cli/app.tsx
  • Web Server: src/web/server.ts
  • FieldView Server: src/web/fieldview-server.ts
  • ACP JSON-RPC: src/acp/server.ts

Section 3: Main Execution Loop(s)

  1. Client sends start_turn to daemon (WebSocket protocol).
  2. Daemon creates/loads session and starts runAgentTurn.
  3. Agent builds system prompt + context (skills, plans, summaries).
  4. Provider streams tokens and tool calls.
  5. Tools execute in parallel, results are appended to messages.
  6. Loop repeats until completion/iteration limit or cancellation.

Repeated batch runs that watch task files, detect stall, and optionally invoke an oracle escalation.

Section 6: Design Patterns & Insights

Patterns Observed

  • Adapter/Strategy: Provider implementations
  • Registry/Plugin: Tool system
  • Event-driven streaming: WebSocket + chunks
  • Pipeline: prompt → model → tools → response

Strengths

  • Clear separation between UI and runtime
  • Tool schema validation + circuit breakers
  • Multi-provider support with config overrides
  • Extensive automation tool catalog

Potential Issues

  • High complexity in agent loop can be hard to debug
  • Multiple UIs with similar protocol mapping paths
  • Tool execution parallelism may race on shared state
  • Heavy reliance on env/config consistency

Improvement Ideas

  • Centralize protocol translation for Ink/Web/ACP to reduce duplication.
  • Introduce typed event bus for subagent + tool telemetry.
  • Add formal contract tests for tool schemas vs implementations.

Section 7: Full Summary

  • Primary execution loop is runAgentTurn streaming provider output and executing tools.
  • Daemon is the hub: WebSocket protocol + session management + tool context.
  • Tool registry is a plugin system with parallel execution and structured logging.
  • Multiple surfaces: CLI, Ink UI, Web UI, FieldView UI, WhatsApp, ACP JSON-RPC.
  • Autonomy + Scheduler enable long-running and scheduled agent workflows.
