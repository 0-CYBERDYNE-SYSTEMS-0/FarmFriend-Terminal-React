Workspace Memory Map

This map frames the workspace as a product surface. It compares ClawdBot's explicit memory files with FF-Terminal's implicit workspace artifacts, then proposes a contract that turns memory into a deployable, inspectable spec.

Memory Surface

ClawdBot Contract (Explicit Files)

AGENTS.md -- agent persona + allowed modes SOUL.md -- behavioral tone and guardrails TOOLS.md -- tool access and operational limits MEMORY.md -- long-lived memory and facts PLAN.md -- active plan snapshot TASKS.md -- task queue + status LOG.md -- audit trail of key actions
Current State

FF-Terminal Workspace (Implicit)

/ff-terminal-workspace /sessions -- chat history per session /logs -- structured logs and autonomy logs /whatsapp -- auth, sessions, allowlist /skills -- skill library + drafts /todos -- session todo lists /projects -- user-defined project spaces /memory_core -- session_summary.md (implicit)
Gap

Why It Matters

  • Explicit files make behavior inspectable and portable across deployments.
  • Implicit memory reduces user control and makes audits harder.
  • A contract enables tooling, reviews, and safer automation.

Proposed Workspace Contract for FF-Terminal

Human-Readable Contract

  • AGENTS.md (persona + tool tiers)
  • TOOLS.md (tool policy + limits)
  • PLAN.md (active plan + progress)
  • MEMORY.md (facts + preferences)

Runtime Artifacts

  • sessions/ (history snapshots)
  • logs/ (audit + errors)
  • whatsapp/ (pairing + allowlist)
  • skills/ (bundled + external)

Implementation Notes

  • Load AGENTS/TOOLS/MEMORY into system prompt at session start.
  • Write PLAN.md on plan updates; expose in CLI UI.
  • Emit LOG.md summaries after each turn or at task end.
  • Make contract optional but discoverable in UI.

Refactor Roadmap

Phase 1: Introduce contract files in workspace wizard.
Phase 2: Wire contract into system prompts and logs.
Phase 3: Build a workspace diff viewer in CLI/Web UI.
Phase 4: Add export/import for deployment portability.

Experience Design Levers

  • Expose a "Memory Map" panel with status indicators for each file.
  • Highlight stale or missing files with warnings.
  • Allow user edits with a structured validation preview.
  • Provide one-click "sync" to regenerate missing artifacts.
