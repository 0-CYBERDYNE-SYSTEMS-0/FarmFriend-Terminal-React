FF‑Terminal User Guide

Natural Language, Maximum Impact

Talk to FF‑Terminal like a smart coworker. Give clear direction, context, and constraints — and let it do the heavy lifting.

The 10‑80‑10 Rule

You do the first 10% (setup + clarity). FF‑Terminal handles 80% (the work). You finish the last 10% (review + polish).

This guide shows how to maximize that 80%.

What “Prompting” Means

Plain English

Prompting is simply asking in natural language with enough detail that FF‑Terminal can act without guessing.

Mental Model

Think of FF‑Terminal as a proactive teammate: you say the goal, give context, define boundaries, and request the output format.

Why It Works

Good prompts remove ambiguity and unlock faster, more accurate results.

The 10‑80‑10 Workflow

10% Setup
80% Execution by FF‑Terminal
10% Review

10% Setup (You)

  • State the goal in one sentence.
  • Provide context: files, audience, constraints.
  • Specify format: bullets, table, JSON, etc.
  • Define “done.”

10% Review (You)

  • Scan for accuracy and tone.
  • Request tweaks or deeper detail.
  • Approve or iterate once more.

Prompt Ingredients (Context Engineering)

Context engineering means giving the right ingredients so FF‑Terminal can cook fast and confidently.

Ingredient Why It Matters Example
Goal Defines success “Draft a 1‑page client email about project status.”
Audience Controls tone and vocabulary “Non‑technical executives.”
Constraints Prevents surprises “Keep it under 150 words; no jargon.”
Inputs Gives facts and sources “Use this bullet list of updates…”
Output format Makes results usable “Return a table with columns X/Y/Z.”
Quality bar Sets expectations “Make it concise, professional, confident.”

Tips, Tricks, and Power Moves

Be Specific

Specify the audience, format, and constraints. Ambiguity steals the 80%.

Feed It Inputs

Give bullet points, notes, or links. FF‑Terminal is strongest when it has raw material.

Ask for Options

“Give me 3 variations.” You’ll get faster comparison and better choices.

Define Done

Tell it exactly what “finished” looks like (word count, format, checks).

Tighten Iteratively

First pass = broad. Second pass = precise. Third pass = ready to ship.

Ask for Risks

“List edge cases or risks I should review.” This catches blind spots.

Do This / Not That

Do This Not That
“Summarize this report for a client, 5 bullets.” “Summarize this.”
“Return a 2‑column table: issue + fix.” “List problems.”
“Use a confident, friendly tone.” “Make it good.”
“Assume audience is non‑technical.” “Explain it.”

Quick Start Script (One Prompt)

Try this: “You’re my assistant. I need [deliverable]. Audience: [who]. Constraints: [time/length]. Inputs: [paste notes]. Output as [format]. Give 2 alternatives.”
