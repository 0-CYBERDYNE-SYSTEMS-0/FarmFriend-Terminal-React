Skill Atlas

A visual inventory of skill system architecture and governance. This compares FF-Terminal's YAML Markdown skills with ClawdBot's AgentSkills + metadata gating model, then highlights upgrade paths for distribution and safety.

Governance

Skill Policy Differences

  • ClawdBot: metadata gates OS, gateway, and binary tools per skill.
  • FF-Terminal: allowed-tools enforced post skill-load per session.
  • Opportunity: add optional metadata for pre-flight compatibility checks.
Distribution

Skill Catalog Strategy

  • ClawdBot: curated catalog and managed local skills store.
  • FF-Terminal: bundled skills + optional external mounts.
  • Opportunity: introduce a registry.json + CLI installer.
Workflow

Authoring Flow

  • FF-Terminal: skill_draft -> skill_apply with assets.
  • ClawdBot: AgentSkills with workspace overrides.
  • Opportunity: add a "skill test harness" for validation.
Safety

Tool and OS Controls

  • ClawdBot: explicit OS level and external binary gates.
  • FF-Terminal: allowed-tools hook for tool registry calls.
  • Opportunity: add a "dangerous tool" label and review workflow.
