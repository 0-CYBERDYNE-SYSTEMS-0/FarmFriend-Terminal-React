Agriculture-First Overhaul Plan

Planning document for a major branch-based overhaul. Scope: adopt a ClawdBot-style contract, stabilize session continuity, and deliver a frictionless agricultural user experience. No code changes in this document.

Goal

Primary Outcome

  • Make the product "just work" for low-tech agriculture users.
  • Establish continuous sessions with stable artifacts.
  • Adopt explicit workspace contract files without redundant storage.
Non-Goals

What We Are Not Doing Yet

  • No multi-channel expansion until gateway abstraction is stable.
  • No public skill marketplace until tool governance is locked down.
  • No heavy UI clutter in the core CLI during MVP overhaul.
Project Strategy

Branch-Based Overhaul

  • Create a dedicated overhaul branch for the team.
  • Keep the current main stable; new flow ships behind a feature flag.
  • Maintain a migration plan for existing workspaces.

Phase Plan

Phase 0 Duplicate repo or create a branch; freeze scope; define success metrics.
Phase 1 Workspace contract: add AGENTS/TOOLS/MEMORY/PLAN/TASKS/LOG files.
Phase 2 Session continuity: enforce canonical workspace + stable session ID.
Phase 3 UX simplification: hide advanced config; WhatsApp-first onboarding.
Phase 4 Governance: tool manifest + skill metadata gates + logging.
Phase 5 Migration + QA: import old data, verify regression tests, ship.

Session Model Target

Current
workspace -> session id -> local artifacts
Target
canonical workspace -> stable session id -> continuous memory
Benefit
consistent assets + no re-explaining + trust continuity

Agriculture-First UX Requirements

  • WhatsApp-first onboarding with zero config.
  • Plain-language prompts and confirmations.
  • Daily check-ins and actionable reminders (weather, tasks, equipment).
  • UI hides advanced features until explicitly requested.

Risk & Mitigation

Risk Redundant memory stores -> Mitigation Single source of truth in MEMORY.md.
Risk Session fragmentation -> Mitigation Canonical workspace + stable session ID.
Risk UX confusion -> Mitigation Hide config, focus on tasks.

Acceptance Criteria

  • User can start via WhatsApp in under 2 minutes.
  • All artifacts appear in a single workspace across projects.
  • AGENTS/TOOLS/MEMORY/PLAN/TASKS/LOG are populated and readable.
  • No duplicated memory or task state conflicts.

Deliverables

  • Migration plan from existing workspaces.
  • Updated onboarding copy and prompts.
  • Control-plane view for channel health and pairing.
  • Release plan and rollback strategy.
