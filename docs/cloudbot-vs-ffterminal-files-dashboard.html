Core Difference

Implicit vs Explicit

  • FF-Terminal stores memory, tasks, and logs in structured folders and JSON.
  • ClawdBot surfaces a human-readable contract as files in workspace.
  • Adopting ClawdBot files creates overlap unless you define a source of truth.
Session Model

Continuous vs Per-Workspace

  • ClawdBot: “one continuous session” mental model.
  • FF-Terminal: sessions scoped to workspace directory.
  • Preference: define a canonical workspace to simulate continuity.

Bottom Line (Simple)

  • If you adopt the ClawdBot contract, designate one file per category as the source of truth.
  • Use FF-Terminal’s structured stores for computation; use ClawdBot files for readability.
  • Continuity is achievable by forcing a single canonical workspace + stable session ID.
