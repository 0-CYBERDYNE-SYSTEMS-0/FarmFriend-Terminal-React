Deep Comparison — ff-terminal E2E Loop vs. Ralph Wiggum Loop

Step‑by‑step breakdown and evaluation. Sources are limited to local code + external references listed below.

Scope & Constraints

This analysis uses local repo code for the E2E loop and external references for Ralph Wiggum. Official plugin README content was not available in this pass; the plugin details below are from a secondary source.

E2E Loop (ff-terminal)

Runtime Test Harness

Script Entry

./scripts/e2e_loop.sh "<profile>" auto
...prompts on stdin...

Core Behavior

Invokes dist/bin/ff-terminal.js run --headless per prompt, reusing a session ID to persist context.

Ralph Wiggum Loop

Autonomous Iteration Pattern

Minimal Loop

while :; do
  cat PROMPT.md | <agent>
done

Productized Variant

Runs Claude Code repeatedly using PROMPT.md + task files, with exit heuristics and rate limits.

Architecture Diagrams

E2E Loop (ff-terminal) stdin prompts │ ▼ ./scripts/e2e_loop.sh │ ▼ node dist/bin/ff-terminal.js run --headless │ ▼ runAgentTurn (tools + hooks + logs) │ ▼ JSONL session logs
Ralph Wiggum Loop PROMPT.md (+ task files) │ ▼ Claude Code execution │ ▼ exit heuristics / completion promise │ ▼ repeat until done or limits

Step‑By‑Step Mapping

Step E2E Loop (ff-terminal) Ralph Wiggum Loop
Input Source stdin lines (one prompt each) PROMPT.md + task files (persistent)
Entry Point Shell script → ff-terminal CLI Bash loop or ralph/plug‑in command
Context Explicit session ID + workspace logs File‑based context + logs
Execution Core runAgentTurn + tool registry Claude Code runtime
Stop Logic Stop hooks when no tool calls Completion promise / exit heuristics
Safety Tool schema validation + circuit breakers Rate limiting + circuit breaker
Observability Session + run JSONL logs Logs + optional tmux monitor

Which Is “Better”?

Best for correctness & integration validation

E2E Loop wins. It exercises your actual runtime, tools, and hooks deterministically.

Best for long‑horizon autonomy

Ralph loop wins. It’s designed to keep iterating with exit heuristics and rate controls.

Efficiency

E2E loop is efficient for short, targeted prompts. Ralph loop is efficient for long‑running progress but can be costly on short tasks.
