Session Continuity & Workspace Topology

This dashboard contrasts long-lived session continuity (as described in your quad-bop flow) with FF-Terminal's workspace-scoped sessions and per-directory artifacts. It highlights where to stabilize paths, assets, and memory across projects.

Session mode: Continuous vs Workspace-scoped Updated: --
Continuity Model

Long-Lived Session (Quad-bop)

  • Single, continuous session carries memory across tasks.
  • Pre-session handshake or staging can gate tools.
  • Artifacts live under stable paths.
Workspace Model

FF-Terminal Default

  • Workspace directory anchors sessions, logs, and artifacts.
  • Different projects can produce different workspaces.
  • Local mode creates a new workspace in the current directory.
Risk

Path Drift

  • Artifact locations vary by workspace root.
  • Asset routes are not globally stable.
  • Memory consistency depends on workspace selection.

Observed Workspace Anatomy

ff-terminal-workspace/ sessions/ logs/ whatsapp/ skills/ todos/ projects/ memory_core/ tasks.json

If you need global continuity across projects, pick one canonical workspace directory and point all launches to it.

Continuity Gaps to Close

Gap 1: Sessions scoped to workspace, not user identity.
Gap 2: Artifact paths differ per directory run.
Gap 3: Project-level memory overwrites or fragments.

Stabilization Options

  • Set a global workspace directory via config or env var.
  • Introduce a "profile workspace" shared across projects.
  • Write a path manifest file (ASSETS.md) with canonical routes.
  • Expose workspace selection in the CLI and Web UI.

Continuity Flow (Side-by-Side)

Quad-bop (assumed)
pre-session -> tool gating -> continuous session -> stable artifacts
FF-Terminal
launch -> resolve workspace -> session id -> artifacts scoped to workspace

If the goal is continuous sessions across repos, use a shared workspace and a single session id or auto-resume logic per user.

Design Prompt: Path Contracts

  • Create a predictable asset root independent of cwd.
  • Store "project registry" under workspace/projects/.
  • Surface project-to-workspace links in the UI.
  • Avoid implicit path inference when session spans multiple repos.
