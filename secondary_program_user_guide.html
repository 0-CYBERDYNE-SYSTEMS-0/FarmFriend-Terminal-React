Program User Guide

Commands, Wizards, Agents, Skills, UI

This guide explains how the FF-Terminal interface works and how to use slash commands, wizards, skills, and agents with confidence.

Quick Start

Type /help any time. If a command does not do what you expect, use the wizard menu: /wizard.

Tip: Use //text to send a literal slash command as text.

UI Map (What You Are Looking At)

Header: status + mode + hints
Transcript: you, assistant, tools, system messages
Panels: help, wizards, logs, plan, thinking
Input line: type your prompt or /command here

Agents and Skills (What They Are)

Agents

Agents are specialized personas with their own system instructions, tool permissions, and max turn limits.

Create them from templates, by filling a form, or by describing what you want.

Skills

Skills are reusable instructions you can trigger as slash commands. Each skill can include triggers, tags, and recommended tools.

Skills live in workspace/skills and can be mounted read-only from other locations.

Custom Commands

Custom commands are shortcuts that expand into a full prompt template.

They live in workspace/commands as Markdown files.

Tools, Tool Chains, and Workflows

Tools

Tools are built-in actions FF-Terminal can call (file I/O, search, execution, web, etc.).

Use /tools to see the full list available in your session.

Tool Chains

A tool chain is a sequence of tool calls that accomplish a multi-step task (read -> edit -> test -> report).

Ask for step-by-step tool usage if you need visibility or approvals.

Workflows

Workflows combine tools + prompts + skills for repeatable outcomes.

Skills often encode workflows so you can reuse them as one command.

How Slash Commands Flow

If the input starts with a slash, FF-Terminal treats it as a command and routes to the correct wizard or action.

Practical Examples

Open the Skills Wizard

/skills

Browse and create reusable skills.

Run a Custom Command

/review src/cli/app.tsx

Expands a command template using $1.

Check Workspace Health

/doctor

Scans your workspace and offers fixes.

Load a Project Context

/init-project

Pick from workspace/projects with FF_PROJECT.md.
