Scaling Guide

Horizontal and vertical scaling strategies

Horizontal Scaling

Deploy multiple instances behind a load balancer for redundancy and capacity.

Nginx Load Balancer

nginx 
upstream ff_cluster {
    least_conn;
    server 10.0.1.10:3000 weight=3;
    server 10.0.1.11:3000 weight=2;
    server 10.0.1.12:3000 weight=1;
}

server {
    listen 80;
    location / {
        proxy_pass http://ff_cluster;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

Session Management

bash 
# Shared session store (Redis)
export FF_SESSION_STORE=redis
export FF_REDIS_HOST=redis.example.com
export FF_REDIS_PORT=6379
export FF_REDIS_PASSWORD=secure_password

Vertical Scaling

  • Memory: Increase to 4GB+ for large workloads
  • CPU: Multi-core with Node.js cluster mode
  • Storage: SSD for database and logs

Kubernetes Deployment

yaml 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ff-terminal
spec:
  replicas: 3
  selector:
    matchLabels:
      app: ff-terminal
  template:
    metadata:
      labels:
        app: ff-terminal
    spec:
      containers:
      - name: ff-terminal
        image: ff-terminal:2.0.0
        ports:
        - containerPort: 3000
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
---
apiVersion: v1
kind: Service
metadata:
  name: ff-terminal-service
spec:
  selector:
    app: ff-terminal
  ports:
  - port: 80
    targetPort: 3000
  type: LoadBalancer

Performance Tuning

bash 
# Increase worker pool size
export FF_WORKERS=4

# Enable response caching
export FF_CACHE_ENABLED=true
export FF_CACHE_TTL=300

# Optimize database connections
export FF_DB_POOL_SIZE=20
