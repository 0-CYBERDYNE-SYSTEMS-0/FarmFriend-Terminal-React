File Structure

Project organization and directory layout

On This Page

    Overview

    FF Terminal is organized into a modular structure that separates concerns between the daemon, CLI, web interface, and shared utilities.

    Root Directory

    text 
    ff-terminal-ts/
├── src/
│   ├── daemon/           # Background daemon process
│   ├── cli/              # Command-line interface
│   ├── web/              # Web interface
│   ├── shared/           # Shared utilities
│   └── types/            # TypeScript type definitions
├── skills/               # Skill definitions (40+ skills)
├── profiles/             # User profile configurations
├── logs/                 # Application logs
├── docs/                 # Documentation (markdown)
├── tests/                # Test suites
└── package.json          # Dependencies and scripts

    Daemon Module

    The daemon is the core process that handles all tool execution and agent operations.

    text 
    src/daemon/
├── daemon.ts            # Main daemon entry point
├── server.ts             # WebSocket server
├── agent/
│   ├── agent.ts          # Agent orchestration logic
│   ├── context.ts        # AsyncLocalStorage context
│   └── loop.ts           # Autonomy loop implementation
├── tools/
│   ├── registry.ts       # Tool registration and discovery
│   ├── executor.ts       # Tool execution engine
│   └── schemas/          # Tool JSON schemas
├── providers/
│   ├── base.ts           # Abstract provider interface
│   ├── openai.ts         # OpenAI provider
│   ├── openrouter.ts     # OpenRouter provider
│   ├── gemini.ts         # Google Gemini provider
│   └── ...
├── sessions/
│   ├── manager.ts        # Session management
│   └── storage.ts        # JSON-based storage
└── skills/
    ├── loader.ts         # Skill loading logic
    └── executor.ts       # Skill execution

    CLI Module

    The CLI provides a terminal-based interface to interact with the daemon via WebSocket.

    text 
    src/cli/
├── cli.ts                # CLI entry point
├── client.ts             # WebSocket client
├── ui/
│   ├── renderer.ts       # Terminal output rendering
│   ├── input.ts          # User input handling
│   └── menus.ts          # Interactive menus
├── commands/
│   ├── chat.ts           # Chat command
│   ├── tool.ts           # Tool execution command
│   ├── skill.ts          # Skill commands
│   └── ...
└── utils/
    ├── formatting.ts     # Output formatting
    └── validation.ts     # Input validation

    Web Module

    The web interface provides a browser-based UI for interacting with FF Terminal.

    text 
    src/web/
├── server.ts             # Express/HTTP server
├── client/
│   ├── index.html        # Main HTML page
│   ├── main.tsx          # React application
│   └── websocket.ts      # WebSocket client
├── components/
│   ├── Chat.tsx          # Chat interface
│   ├── Terminal.tsx      # Terminal display
│   └── Sidebar.tsx       # Navigation
└── api/
    ├── routes.ts         # API endpoints
    └── middleware.ts     # Express middleware

    Shared Utilities

    Shared code used across daemon, CLI, and web interface.

    text 
    src/shared/
├── config.ts             # Configuration management
├── logger.ts             # Logging utilities
├── errors.ts             # Error classes
├── validation.ts         # Validation schemas
└── constants.ts          # Application constants

    Skills Directory

    Each skill is a self-contained module with its own SKILL.md, code, and tests.

    text 
    skills/
├── design/
│   ├── canvas-design/
│   ├── award_winning_designer/
│   └── ...
├── development/
│   ├── component_library/
│   ├── algorithmic-art/
│   └── ...
├── media/
│   ├── remotion-expert/
│   ├── slack-gif-creator/
│   └── ...
└── ...

Each skill:
├── SKILL.md              # Skill documentation
├── index.ts              # Skill implementation
├── test/                 # Skill tests
└── tools/                # Skill-specific tools

    Key Design Decisions

    • Separation of Concerns: Daemon handles execution, CLI/web handle UI
    • Modular Skills: Each skill is independent and can be loaded/unloaded
    • Type Safety: TypeScript throughout with strict type checking
    • Shared Code: Common utilities extracted to src/shared/
    • Test Isolation: Tests co-located with source code
