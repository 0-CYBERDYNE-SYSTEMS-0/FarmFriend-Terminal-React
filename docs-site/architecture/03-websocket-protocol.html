WebSocket Protocol

Real-time bidirectional communication between daemon and clients

Overview

FF Terminal uses WebSocket for real-time, low-latency communication between the daemon and its clients (CLI, web). The protocol supports message streaming, command execution, and event notifications.

Connection

javascript 
const ws = new WebSocket('ws://localhost:28888');

ws.onopen = () => {
  console.log('Connected to FF Terminal daemon');
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  handleMessage(message);
};

Message Types

Client → Daemon

Type Description
tool_execution Execute a tool with arguments
chat Send user message to agent
skill Execute a skill
config Update configuration
session Session management commands

Daemon → Client

Type Description
chunk Streamed content chunk
complete Execution completed successfully
error Execution failed
status Daemon status update
event Daemon event notification

Message Format

typescript 
// Base message interface
interface Message {
  id: string;           // Unique message ID
  timestamp: number;    // Unix timestamp
  sessionId: string;    // Session UUID
  type: MessageType;    // Message type
}

// Tool execution request
interface ToolExecutionMessage extends Message {
  type: 'tool_execution';
  tool: string;         // Tool name
  args: Record; // Tool arguments
  stream?: boolean;     // Enable streaming (default: true)
}

// Streaming response chunk
interface ChunkMessage extends Message {
  type: 'chunk';
  content: string;      // Chunk content
  progress?: number;    // Optional progress (0-1)
}

// Complete response
interface CompleteMessage extends Message {
  type: 'complete';
  result: any;          // Final result
  metrics?: {           // Optional metrics
    duration: number;
    tokensUsed?: number;
  };
}

// Error response
interface ErrorMessage extends Message {
  type: 'error';
  error: {
    code: string;        // Error code
    message: string;    // Error message
    details?: any;       // Additional details
  };
}

Streaming Example

typescript 
// Client sends request
ws.send(JSON.stringify({
  id: generateId(),
  sessionId: 'abc-123',
  type: 'tool_execution',
  tool: 'run_command',
  args: { command: 'npm test' }
}));

// Daemon streams chunks
// Message 1: chunk
{ "id": "...", "type": "chunk", "content": "Running tests..." }
// Message 2: chunk
{ "id": "...", "type": "chunk", "content": "✓ test/unit/cli.test.ts" }
// Message 3: chunk
{ "id": "...", "type": "chunk", "content": "✓ test/unit/agent.test.ts" }
// Message 4: complete
{ "id": "...", "type": "complete", "result": { "exitCode": 0 } }

Error Handling

The daemon sends structured error responses for failures:

json 
{
  "id": "msg-123",
  "type": "error",
  "error": {
    "code": "TOOL_NOT_FOUND",
    "message": "Tool 'invalid_tool' not found",
    "details": {
      "availableTools": ["read_file", "write_file", "..."]
    }
  }
}

Heartbeat

Clients send periodic heartbeat messages to keep connection alive:

json 
{
  "type": "heartbeat",
  "timestamp": 1704235200000
}
